# Site Audit: 30 Concrete Improvements

These are the 30 issues reviewed across the homepage, Platform, Workflow, Research, Contact, shared modules, and repeated card patterns. The matching implementation is in the final override block of `app/globals.css`.

## Global Layout

1. The top navigation still reads as a small floating widget; make it wider, flatter, and more like site chrome.
2. The global content stage is inconsistent; normalize all main pages to a wide 1760px stage.
3. Page sections feel like stacked template blocks; use larger vertical rhythm and fewer framed areas.
4. The visual system has too many competing card/background rules from previous iterations; add a final canonical override.
5. Cards are overused for conceptual content; convert conceptual cards to line-based editorial modules.

## Homepage

6. The hero needs stronger spatial hierarchy; keep the perspective field but make copy and visual layer separation clearer.
7. The brand manifesto needs more editorial weight; it should behave like a core statement, not a normal section.
8. The architecture section should feel like a system view; rows should be precise and less card-like.
9. Product lines should read as three serious business lines, not feature cards.
10. The proof statement should feel like a brand principle, not another card section.

## Platform Page

11. PageLead facts look like small cards; convert them into metric-style inline facts.
12. Platform offerings are conceptually separate but should not look like SaaS feature cards.
13. Platform architecture should become a clear layer list with strong numbering.
14. Product principles should read as rules, not cards.
15. Work surfaces should be organized as operational rows instead of boxed cards.
16. Capability items should be dense, scannable capability lines.

## Workflow Page

17. Workflow steps should read as a vertical operating chain, not independent widgets.
18. Step numbers need stronger alignment and less pill styling.
19. Artifact cards should become simple deliverable rows.
20. The handoff section should look like an executive decision block.

## Research Page

21. The thesis section should read like a publication-quality argument.
22. Research themes should be structured as rows, not cards.
23. Evaluation items should be lighter and more analytical.
24. Narrative text needs a better longform reading width and larger type.

## Contact Page

25. The contact primary area should be calmer and more official.
26. Contact details should look like formal contact metadata, not cards.
27. Inquiry types should be clear rows.
28. Contact steps should feel like a process, not a card grid.

## Shared UX

29. Mobile should not collapse into a pile of cards; rows and dividers should remain.
30. Hover effects should be restrained; professional sites should not bounce every module.
